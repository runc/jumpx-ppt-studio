# Claude Design 操作手册 · 从零上下文到产出生产代码

> 给操作者(你)在 Claude Design 里一步步做出我们 AI Slides WebApp 的**专属界面**,并把代码交回给 Claude Code(我)集成进 Next.js 前端、接上 deepagents 后端。
> 设计依据见 [`UX_DESIGN.md`](UX_DESIGN.md)。
> ⚠️ Claude Design 为 2026-04 发布、在我知识截止后,本手册指引的是**内容与流程**(粘贴什么、按什么顺序、产出什么);具体按钮以你看到的实际界面为准。

---

## 0. 先记住三件事

1. **我们要 Claude Design 做的是「App 界面外壳」**(画布、缩略图轨、模板网格、大纲编辑器、agent 副驾面板),**不是幻灯片成品**。幻灯片成品由后端 skill 生成(teaching-clean 那套)。两套设计系统分开:
   - **App 外壳**：工具型、克制、内容优先(本手册帮你定)。
   - **幻灯片内容**：teaching-clean(Inter,见下),作为 App 里被预览的对象。
2. **目标产物**：可交付给我的**前端代码**(最好 React + Tailwind + shadcn/ui,因为现有 deep-agents-ui 就是 Next.js/React/Tailwind 栈),外加你的**交互观察笔记**。
3. **顺带学习**:Claude Design 自己怎么做"对话改设计 / inline 批注 / 实时调节滑块 / 导出 handoff"——这正是我们产品要借鉴的交互,边用边记(见 §7 观察清单)。

## 阶段A · 播种上下文：开局粘这段 Master Brief

> 在 Claude Design 新建项目,第一条就粘这段。它把"我们是什么、给谁、要哪几屏、什么基调、输出什么"一次讲清。

```
我在做一个 AI Slides（AI 幻灯片）生成 WebApp 的前端界面。请帮我设计这个 App 的界面外壳（不是幻灯片成品本身）。

【产品一句话】
用户输入一个主题，一个自主 AI agent 自动规划并生成一份幻灯片；过程中在几个关键岔路口停下来让用户做选择；最终产出可翻页的幻灯片，可导出。

【用户】中文为主的训练营学员，非设计师，要快、要能看懂、要能交作业。界面文案用简体中文。

【核心理念（重要）】
这是一个「画布为主、AI 为副驾」的工具，不是聊天框。幻灯片画布是绝对主角，AI 对话面板是辅助。请避免“又一个紫色渐变 SaaS 聊天界面”的通用 AI 外观。

【整体布局】三栏 + 顶栏：
- 顶栏：项目名 + 一条阶段进度条（输入 → 大纲 → 选模板 → 渲染 → 完成）
- 左：幻灯片缩略图轨（每页一个小卡，可点跳转；生成时先骨架屏再逐页填充）
- 中：主舞台 = 当前页大图实时预览（所见即所得，键盘左右翻页）
- 右：Agent 副驾面板 = ①三态待办清单(待办/进行中/完成) ②对话框 ③可展开的“它在干什么”活动流

【我需要你逐屏帮我设计这 5 个界面】（后续我会一屏一屏给你细节）：
1. 主工作台（上面的三栏布局，渲染进行中的状态）
2. 输入起步页（多入口：主题词 / 粘贴大纲 / 上传讲稿）
3. 大纲编辑器（左大纲树 / 右线框预览）
4. 选模板（缩略图网格，7 套风格，推荐项预选）
5. 选输出形态（HTML 网页 vs AI 配图，二选一卡片）

【基调】克制、清晰、内容优先、留白充足、专业工具感；浅色为主、深色可选；中文排版友好。先别用 Inter/Roboto 这类被用烂的字体，给一个有气质又适合中文界面的字体方向。

请先理解，然后从【主工作台】开始设计。我们一屏一屏来。
```

## 阶段B · 建立 App 设计系统

Claude Design 在 onboarding 时可读代码库/设计文件自动建 design system。两条路任选:

- **省事**：让它根据上面 Master Brief 自己提一套 App 外壳的 design system，你审色板/字体/圆角/间距基调即可。
- **更可控**：把下面这段我们的取向喂给它(注意这是 **App 外壳**,不是幻灯片的 teaching-clean)：

```
App 外壳设计系统取向：
- 性格：工具型、冷静、内容优先；不要营销味、不要花哨渐变。
- 中性灰阶打底（界面背景近白、文字深灰），一个克制的强调色用于“当前/选中/主操作”。
- 字体：界面用一套适合中英混排、清晰专业的无衬线（避开 Inter/Roboto/Arial）；正文行高宽松。
- 圆角中等、阴影轻、分隔线浅；信息密度中低，多留白。
- 必须支持深色模式。
- 注意：这套是“App 界面”的系统。被预览的“幻灯片内容”另有一套样式(teaching-clean)，画布里的幻灯片应看起来像“成品文档”，与 App 外壳形成主-客对比。
```

> 供参考——**幻灯片内容**(画布里被预览的对象)的真实 token,你在做"主舞台/缩略图"时让它照这个渲染示意页：
> - 背景 `#F8FAFC` / 主色(文字)`#111827` / 强调 `#2563EB` / 次要文字 `#64748B` / 描边 `#D7DEE8`
> - 16:9 画布；标题/正文 `Inter, "Noto Sans SC"`；留白多、不塞字。

## 阶段C · 逐屏生成（每段单独粘，生成后用它的工具迭代）

> 顺序建议：先 1 主工作台(定骨架)→ 4 选模板 + 5 选输出(我们最违和、最该先看到的两个触点)→ 3 大纲 → 2 输入。

### 屏 1 · 主工作台（三栏，渲染中状态）
```
设计【主工作台】，状态为“正在渲染第 3/5 页”。三栏 + 顶栏：
- 顶栏左：项目名“远程工作的三个高效习惯”；中：阶段进度条，高亮“渲染”。
- 左栏：5 个幻灯片缩略图竖排，前 2 个已填充内容、第 3 个进度态、第 4-5 个骨架屏占位；hover 缩略图浮现小操作（重做这页/换图/换布局）。
- 中栏主舞台：第 3 页的大图实时预览（一张 teaching-clean 风格的对比页示意），底部页码 3/5、左右翻页箭头。
- 右栏 Agent 副驾：上半是三态待办清单（“写大纲 ✓ / 选模板 ✓ / 生成页面 ⏳ / 导出 ◻”）；下半是对话框；中间一行可展开“正在套用 teaching-clean，渲染第 3 页…”。
- 底部一个醒目的【停止】按钮（生成可中断）。
中文界面。强调“画布是主角、右侧 AI 是副驾”。
```

### 屏 4 · 选模板（缩略图网格）—— 最关键
```
设计【选模板】界面，它在主舞台中央就地浮现（不是突兀弹窗，像 AI 递来的一步）。
- 顶部一行：“AI 推荐：teaching-clean（已选）/ sketch-notes / corporate”，附一句推荐理由。
- 下方 7 个模板卡缩略图网格：teaching-clean(已选,高亮边框+对勾)、editorial-magazine、swiss-system、blueprint、sketch-notes、corporate、creator-social。
- 关键：每个缩略图都用“用户自己的封面内容”渲染成该风格的封面小图（不是通用占位图），让用户所见即所得地对比。
- 底部主按钮：“用 teaching-clean”（默认已选，可一键沿用）。
- 一句副文案解释“选模板=选整体视觉风格，之后还能一键换肤不动内容”。
中文界面，克制专业。
```

### 屏 5 · 选输出形态（二选一卡片）
```
设计【选择输出形态】，主舞台中央两张并排大卡片二选一：
- 卡A “🌐 HTML 网页幻灯片”：秒级生成、可直接翻页、适合快速交付/在线分享；标“默认”，预选高亮。
- 卡B “🖼️ AI 配图幻灯片”：每页 AI 出图(约 60-90 秒/页)、视觉冲击强、适合封面金句；标注“⚠ 需配置图片 API”。
- 每张卡有一张示意缩略图、3 条要点、一个选择按钮。
- 顶部一行小字解释“为什么问你这个”。
中文界面。
```

### 屏 3 · 大纲编辑器（左树右预览）
```
设计【大纲编辑器】，左右分栏：
- 左：可编辑的大纲树（支持 Markdown 层级），示例内容“# 远程工作的三个高效习惯 / ## 习惯一：固定仪式 / - 启动·关闭工作 / ## 习惯二&三 / ## 今天就开始”；hover 章节出 [+添加][删除][↑↓调序]。
- 右：低保真线框预览，每个章节对应一个灰模幻灯片框（封面/要点/对比/总结），与左侧实时同步。
- 右下角两个按钮：“💬 让 AI 改大纲”、主按钮“✓ 确认大纲，继续”。
- 顶部一句引导：“现在改大纲很快；渲染成页后再改成本更高。”
中文界面。
```

### 屏 2 · 输入起步页（多入口）
```
设计【输入起步页】：一个居中的大输入框 + 三个并排入口卡：
- “💡 给个主题”（默认，主输入框）
- “📝 粘贴大纲”（支持 Markdown）
- “📄 上传讲稿/文档”
下方一行可编辑的“推断设置”小chip：受众、页数(默认8-12)、场景，可点改。
主按钮“开始生成”。
干净、欢迎感、低门槛，中文界面。
```

## 阶段D · 迭代：用 Claude Design 的工具改

每屏生成后,用它的能力打磨(这些操作同时是你要**学习**的范式)：
- **对话改**：“把右侧 Agent 面板再窄一点”“缩略图换成横向排在底部试试”。
- **inline 批注**：直接在元素上标注要改什么。
- **调节滑块**：用它自动给出的间距/圆角/配色滑块微调。
- 让它出**2-3 个变体**对比（学它怎么做多方案）。

## 阶段E · 导出代码 + 笔记，交回给我

### 要导出的东西
1. **代码**：优先导出 **React + Tailwind**（最好含 shadcn/ui）。若只能导 HTML/CSS 也行，我来移植。每个屏都导。
2. 若有 **“handoff 到 Claude Code”** 选项：用它打包(它就是给 Claude Code 吃的)。
3. **设计 token**：导出/记录最终的色板、字体、圆角、间距、阴影值。

### 交回给我时,请一并带上（决定集成质量）
- [ ] 各屏代码（贴进对话，或存到 `ui-design/` 目录我去读）。
- [ ] 各屏**截图**（我看不到你屏幕，截图帮我对齐）。
- [ ] 最终 **design token** 清单。
- [ ] 你的**交互观察笔记**（§7 清单的答案）——我们要把好用的范式落进自己产品。
- [ ] 你最满意/最不满意的地方，要我重点保留或改的点。

### 一句话交接模板（你回来对我说）
> “Claude Design 做完了 5 个屏,代码在 X / 截图在 Y / token 如下…… 我最想保留的交互是 ___,帮我集成进 Next.js 前端并接上 deepagents。”

## 阶段F · 我接手（回到 Claude Code）
拿到你的产物后,我会：
1. 在 `ui/`(或新建轻量前端)落地这些组件,套用导出的 design token。
2. 把 `choose_template` / `choose_render_mode` 的 interrupt payload 结构化,前端按 tool 名渲染你设计的模板网格/输出卡片(替掉现在的 JSON 审批面板)。
3. 把 outline 确认改成你设计的大纲编辑器。
4. 接 LangGraph server 流式：缩略图逐页填充、三态待办、可中断。
5. `/verify` + 预览迭代,直到和你在 Claude Design 里看到的一致。

## 7. 学习清单：用 Claude Design 时重点观察这些（我们产品要借鉴）

边操作边记下答案——这些正是我们 AI PPT 产品自己的交互要解决的同类问题：
1. 它怎么让你**改一处而不破坏整体**？(对应我们"改一页不串其他页")
2. **inline 批注**的触发与呈现：怎么在元素上标注、AI 怎么回应？
3. **调节滑块**：它对哪些属性给了实时滑块(间距/色/圆角)？滑块比打字好在哪？
4. 它**一次给几个方案**？怎么对比、怎么选、怎么"基于这个再改"？
5. **等待/生成**时的反馈：进度?逐步浮现?可打断?
6. **导出/handoff** 的体验：从设计到代码这一步它怎么做的、缺什么。
7. 哪一步让你觉得"**这个交互真顺**"或"**这里很烦**"——记下来,对应到我们的 4 个触点。

## 附：常见卡点
- **它默认做成"通用 AI 界面"**(紫渐变/卡片堆)→ 明确说"工具型、内容优先、避开 Inter、画布为主"。
- **它把"幻灯片内容"和"App 外壳"混了** → 强调"我要的是承载幻灯片的工具界面,幻灯片是被预览的对象"。
- **导不出 React** → 退而导 HTML/CSS + 截图,我来移植成 React 组件。
- **想不起要哪几屏** → 回到 §阶段C 的 5 段 brief。
